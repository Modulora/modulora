## Why everything is reviewed

Nothing lists publicly until a curator approves it. This is the trust spine
of the registry: automated checks prove *provenance* (who published, exactly
which bytes install, whether secrets leaked), but a human decides whether a
submission is real, functional, and honestly described. Approval is a
listing decision — it is **not** a code audit, and the evidence panel on
every component says exactly what was and wasn't checked.

## What curators review

Each submission in **Dashboard → Curation → Review queue** shows the source,
the live demo, and the automated evidence. Curators check:

- **It's real** — the component renders, the demo demonstrates it, the
  title/description match what the code does.
- **It's theirs to publish** — no lifted source misrepresented as original.
  The optional *Original URL* and *Inspired by* links exist for honest
  attribution; missing attribution for obviously derived work is a reject.
- **It's safe to list** — no malicious patterns, no misleading install
  behavior. The secret scan and digest checks run automatically; curators
  read the code for what scanners can't see.
- **Paid listings are honest** — external components must link to a valid
  purchase URL. During alpha, domain verification is optional, so an
  unverified destination must be disclosed and the listing must not imply
  Modulora reviewed source it never saw.

Rejections always carry a reason — the creator sees it verbatim on their
dashboard and by email, fixes, and resubmits. Every new release of an
approved component re-enters the queue.

## What curators can never do

Curation is deliberately separated from money:

- Curators review content; they **cannot** run payout distributions, change
  prices, or see anyone's earnings.
- Approval cannot be bought. Promotion is paid *placement*, marked as such,
  and has no effect on review outcomes or ordering within the queue.
- Plus subscriptions never buy review priority.

## How roles are assigned

- **Curator** — assigned by an owner in **Dashboard → Platform → Admin →
  Roles**, by username. Curators get the Review queue in their dashboard.
- **Owner** — configured only by the `OWNER_USER_IDS` environment variable
  (user IDs, never emails). No code path grants ownership from the UI, so a
  compromised account can't mint owners. Owners run payout distributions
  and assign curators.

Operators self-hosting the registry set `OWNER_USER_IDS` at deploy time;
everything else is managed from the Admin page.
