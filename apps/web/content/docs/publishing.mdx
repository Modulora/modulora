## Build

Create a component at **Dashboard → New component**. The editor gives you a
file rail, a code editor, and a live preview running in an isolated sandbox.
Three file roles:

- **Component** — what installs into a user's project (`components/ui/…`).
- **Demo** — preview-only; its default export renders in the live preview.
- **System** — `package.json`, `index.css`, `lib/utils.ts` (editable, not shipped).

Only component-role files ship in the install payload — and only they are
covered by the content digest.

## From the terminal

The CLI publishes through the same pipeline — same validation, same review:

```bash
npx modulora login
npx modulora publish            # reads ./registry-item.json
```

The manifest lists your files (contents load from disk); paths under
`src/demos/` are preview-only, `src/components/**` is what installs:

```json
{
  "name": "cli-badge",
  "title": "CLI Badge",
  "description": "A small badge component.",
  "category": "data-display",
  "files": [
    { "path": "src/components/ui/cli-badge.tsx" },
    { "path": "src/demos/default.tsx" }
  ]
}
```

Publishing requires accepting the [publishing policy](/publishing-policy) —
interactively, or with `--accept-policy` in CI.

## Review

Every submission goes through curator review before it lists publicly.
You'll see the status on your components: **In review**, **Live**, or
**Changes requested** (with the reason). Review checks the listing is what it
claims to be; it is not a safety guarantee.

## Sell externally (optional)

Direct checkout through Modulora isn't live during alpha. You can link a
commercial listing to your own purchase page. Domain verification is optional
during alpha; unverified destinations are labeled on the public listing. You
handle checkout, licensing, delivery, refunds, and support; Modulora hosts no
paid source, records no purchase, and takes no fee.
