## Build

Create a component at **Dashboard → New component**. The editor gives you a
file rail, a code editor, and a live preview running in an isolated sandbox.
Three file roles:

- **Component** — what installs into a user's project (`components/ui/…`).
- **Demo** — preview-only; its default export renders in the live preview.
- **System** — `package.json`, `index.css`, `lib/utils.ts` (editable, not shipped).

Only component-role files ship in the install payload — and only they are
covered by the content digest.

## From the terminal

The CLI publishes through the same pipeline — same validation, same review:

```bash
npx modulora login
npx modulora publish            # reads ./registry-item.json
```

The manifest lists your files (contents load from disk); paths under
`src/demos/` are preview-only, `src/components/**` is what installs:

```json
{
  "name": "cli-badge",
  "title": "CLI Badge",
  "description": "A small badge component.",
  "category": "data-display",
  "files": [
    { "path": "src/components/ui/cli-badge.tsx" },
    { "path": "src/demos/default.tsx" }
  ]
}
```

Publishing requires accepting the [publishing policy](/publishing-policy) —
interactively, or with `--accept-policy` in CI.

## List an external tool or site

Use **Dashboard → New → List a tool/site** for an owner-authorized external
resource. You must verify the site's exact domain before Modulora fetches its
metadata or accepts the submission.

Modulora fetches the page server-side, mirrors a valid Open Graph image into
its media storage, and prepares an isolated iframe preview. The iframe uses an
opaque sandbox without same-origin access. Sites that block embedding retain
the mirrored Open Graph image as a fallback.

Tool and site listings are not installable components. Their detail pages show
the live preview, description, category, scoped domain-control evidence, and a
link to the verified site.

## Review

Every submission goes through curator review before it lists publicly.
You'll see the status on your listings: **In review**, **Live**, or
**Changes requested** (with the reason). Review checks the listing is what it
claims to be; it is not a safety guarantee.

## List a tool or site

Choose **New → Tool or site** to submit an owner-authorized external resource.
Before submitting, verify the exact domain in **Settings → Verified domains**,
inspect the HTTPS URL, and provide:

- a title, description, catalog subcategory, and listing slug;
- an external pricing label: **Free**, **Freemium**, or **Paid**; and
- 1–6 creator-owned PNG, JPEG, or WebP site thumbnails, up to 2 MB each.

Reorder thumbnails to choose the catalog cover. The full set appears as a
carousel on catalog and detail surfaces. Pricing is descriptive metadata for
the external site; it does not enable Modulora checkout or buyer entitlements.
Tool listings also appear under the top-level **Tools** catalog category while
retaining their more specific subcategory.

## Sell externally (optional)

Direct checkout through Modulora isn't live during alpha. You can link a
commercial listing to your own purchase page. Domain verification is optional
during alpha; unverified destinations are labeled on the public listing. You
handle checkout, licensing, delivery, refunds, and support; Modulora hosts no
paid source, records no purchase, and takes no fee.
