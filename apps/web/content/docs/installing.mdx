## With the Modulora CLI (verified)

```bash
npx modulora add @creator/component
```

The CLI fetches the release, **recomputes its content digest locally and
compares it to the published digest**, shows a write plan, then writes the
files. It never runs an install script.

Verification outcomes:

- **verified** — the received files match the published digest exactly.
- **mismatch** — they don't; the CLI refuses to write (`--force` overrides,
  don't use it unless you understand the risk).
- **unverifiable** — the release has no published digest; the CLI asks first.

Useful flags: `--dry-run` (plan only), `--yes` (skip overwrite prompts),
`--json` (machine-readable, for agents and CI), `--cwd <dir>`.

## Collections

Creators can group components into installable kits. One command installs
every member — and **each component is digest-verified individually**, never
as a combined blob:

```bash
npx modulora add @creator/dashboard-kit
```

Every member lands in `modulora.lock` with its own version and digest, so
`verify`, `diff`, and `update` work per component afterward.

## External commercial components

Direct purchases through Modulora aren't available during alpha. A commercial
listing can link to the creator's purchase page, but the creator handles
checkout, licensing, delivery, updates, refunds, and support. Domain ownership
isn't required during alpha; each listing states whether Modulora has verified
the destination.

Modulora doesn't host or install creator-fulfilled paid source. It doesn't
record the purchase or assess source it never receives. Follow the creator's
delivery instructions and review their license before adding files to a
project.

## With the shadcn CLI

Every free component also works with shadcn:

```bash
npx shadcn@latest add https://modulora.dev/r/@creator/component
```

Note the shadcn path doesn't verify the digest — the Modulora CLI is the
verified route.

### Installed with shadcn? Verify it after the fact

Both CLIs receive **identical bytes** — `/r/` serves one payload. So a
shadcn install can be verified post-hoc:

```bash
npx modulora check @creator/component          # compare disk ↔ published release
npx modulora check @creator/component --save   # adopt into modulora.lock
```

`check` verifies the upstream digest first, then compares your on-disk
files byte-for-byte. `--save` records a clean result so `verify`, `diff`,
and `update` cover the component from then on.
