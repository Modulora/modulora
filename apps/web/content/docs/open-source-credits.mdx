Modulora stands on open-source work maintained by people and teams across the
web platform. This page records every direct production dependency declared by
the web application, its license, and the role it plays in Modulora.

Versions come from `apps/web/package.json` and the lockfile. License identifiers
come from the installed package manifests through `pnpm licenses list --prod`.
Transitive packages retain their own licenses and notices.

> [!NOTE]
> Modulora's public core is `AGPL-3.0-only`, but that does not replace or alter
> the licenses of the projects below. Creator components also retain their own
> declared licenses. See the repository's
> [licensing policy](https://github.com/Modulora/modulora/blob/main/LICENSING.md)
> for the full boundary.

## Keeping this page accurate

This page covers direct production dependencies, not every package in the
transitive dependency graph. When a direct dependency changes, maintainers run
`pnpm --filter web docs:credits:check`. The check fails when a package is
missing or when its displayed license does not match the installed manifest.

For the complete machine-generated inventory at any commit, run:

```bash
pnpm licenses list --prod
```

Thank you to every maintainer and contributor whose work makes Modulora
possible.
